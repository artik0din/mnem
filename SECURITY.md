# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in Mnem, please do **not** open a public GitHub issue.

Instead, send a report by email to the maintainer or by opening a [private security advisory](https://github.com/artik0din/mnem/security/advisories/new) on GitHub.

When reporting, please include:

- A clear description of the issue and its potential impact.
- Steps to reproduce.
- Any proof-of-concept code.
- Your preferred contact method for follow-up.

We aim to acknowledge reports within 72 hours and will keep you informed as we investigate. Once a fix is available, we will credit you in the release notes unless you prefer to remain anonymous.

## Supported versions

Security fixes are backported to the latest minor release only. Users on older versions are encouraged to upgrade.

| Version | Supported |
| ------- | --------- |
| 0.x     | Yes       |
